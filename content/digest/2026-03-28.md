### Telnyx PyPI 침해 — TeamPCP 캠페인 계속
[Aikido Security 블로그](https://www.aikido.dev/blog/telnyx-pypi-compromised-teampcp-canisterworm)

동일한 TeamPCP 캠페인의 연속으로, 공식 Telnyx Python SDK(월 74.2만 다운로드)의 두 버전(4.87.1, 4.87.2)이 3월 27일에 악성 코드로 업로드되었다. `import telnyx`만으로 실행되며, WAV 파일 스테가노그래피로 페이로드를 숨겨 MIME 타입 체크를 통과하는 새로운 기법이 사용되었다. Windows에서는 XOR 난독화 실행 파일을 Startup 폴더에 드롭하고, Linux/Mac에서는 AES-256-CBC + RSA-4096으로 암호화하여 유출한다.

**TeamPCP 캠페인 타임라인**: 3/19 Trivy 침해(CVSS 9.4) → 3/20 npm CanisterWorm(46+ 패키지) → 3/22 WAV 스테가노그래피 K8s 와이퍼 → 3/23 Checkmarx GitHub Actions 침해 → 3/24 LiteLLM PyPI 침해 → 3/27 Telnyx PyPI 침해.

**즉시 실행 항목**: `telnyx>=4.87.1` 즉시 제거, `telnyx==4.87.0`으로 고정. 감염 환경은 모든 API 키, DB 자격 증명, SSH 키 즉시 교체.

### SHA-256의 92%를 깨뜨렸다
[State of Utopia](https://stateofutopia.com/papers/2/we-broke-92-percent-of-sha-256.html)

전체 64라운드에 걸쳐 SHA-256의 64개 스케줄 방정식 중 59개(92%)를 만족시키는 semi-free-start 충돌을 달성했다는 연구 결과가 발표되었다. 완전한 충돌 공격은 아니지만, 전체 64라운드를 공격하면서 92% 스케줄 준수를 달성한 것은 전례 없는 성과다. 복합적 분석 추론과 저수준 C 프로그래밍의 조합으로, 1,950개의 형식적으로 검증된 Lean 정리를 수집했다. 저자들은 완전한 충돌이 곧 찾아질 것으로 예상하며, 다른 해시 함수 패밀리로의 마이그레이션을 권고한다. 비트코인의 이중 SHA-256 파이프라인에 대한 이론적 영향 가능성도 언급되었다.

## 에이전트 스킬과 워크플로우 혁신

### 스킬에도 TDD가 왔다 — Anthropic skill-creator eval 프레임워크 분석
[LinkedIn - Goobong Jeong](https://www.linkedin.com/feed/update/urn:li:activity:7435085433027469313/)

Anthropic이 skill-creator에 테스트(eval) 프레임워크를 추가하면서, 스킬을 **Capability Uplift**(모델이 못 하는 걸 보완, 모델 발전 시 자연 소멸)와 **Encoded Preference**(팀의 프로세스/방식을 담은 것, 모델이 아무리 좋아져도 남음) 두 종류로 데이터 기반으로 구분할 수 있게 되었다. `run_eval.py`는 스킬의 description을 임시 등록한 뒤 쿼리당 3회씩 병렬 10개 프로세스로 trigger rate를 산출하고, `improve_description.py`가 description을 자동 개선한다. Anthropic 블로그의 핵심 문장: "Eventually, that description may be the skill itself." 구현 방법이 아닌 원하는 결과만 기술하면 되는 방향, 즉 스킬이 코드에서 스펙으로 진화하는 것이 핵심이다.

### Spec 파일 3개부터 만드세요 — Claude Code와 Codex에 일 시키기 전에
[LinkedIn - Jeongmin Lee](https://www.linkedin.com/feed/update/urn:li:activity:7443483254831288320/)

AI 에이전트의 결과물 품질이 매번 달라지는 원인은 "의도를 전달하는 구조"의 부재였다. **3계층 스펙 파일 구조**: Layer 1 — `CLAUDE.md`(프로젝트 헌법), Layer 2 — `SPEC.md`(특정 기능의 what과 why, 목적/요구사항/성공 기준/기술적 제약/경계선), Layer 3 — `plan.md`(2~5분 단위 태스크 리스트). "Curse of Instructions" 연구에 따르면 지시사항이 많아질수록 LLM의 각 지시 준수율이 급격히 떨어진다. 세션 1에서 SPEC.md → 세션 2에서 plan.md → 세션 3부터 태스크 실행으로 분리하니 정확도가 체감될 만큼 올라갔다. Martin Fowler 팀의 에이전트 기반 소프트웨어 개발 성숙도 모델도 소개: Level 1(스펙 쓰고 폐기), Level 2(살아있는 문서), Level 3(스펙이 유일한 진실의 원천, 코드는 생성물).

## 교차 분석

**1. AI의 이중성 — 공격과 방어의 동시 진화**: TeamPCP 캠페인이 LiteLLM과 Telnyx를 체계적으로 공격하는 동시에, Claude Code가 72분 만에 악성코드를 탐지/분석/공개하는 데 핵심 역할을 했다. Linux 커널에서도 AI 버그 리포트가 "쓸모없는 것에서 합법적인 것으로" 변곡점을 맞이했다. AI가 보안의 양면에서 동시에 역할을 확대하고 있으며, 이는 AI 생태계를 노린 공급망 공격과 AI로 방어하는 구도가 고착화되고 있음을 의미한다.

**2. 파워유저와 일반 사용자의 격차 심화**: Jakob Nielsen의 P95/P50 분석(ChatGPT에서 35배 차이)과 Zoon Chang의 "원래 에이스였던 사람이 토큰도 많이 쓴다"는 관찰, Boris Cherny의 10~15개 병렬 세션 운영, auto-research로 0점이 100점이 되는 실험이 모두 같은 현상을 가리킨다. AI 도구의 접근성은 평등해졌지만, 활용 깊이의 격차는 오히려 더 벌어지고 있다. 이는 SaaS 가격 전략(P95/P50 비율 20배 이상이면 종량제)과 Claude Code의 피크 시간 쿼터 정책까지 연결된다.

**3. 하드웨어와 소프트웨어의 동시 구조 변환**: AI 데이터센터가 하드웨어 공급을 흡수하면서 소비자 하드웨어가 밀려나는 동시에(Mac Pro 단종, PC 가격 15-20% 인상), ATLAS 같은 프로젝트가 $500 GPU로 프론티어 모델에 근접하는 성능을 보여주고 있다. HP의 노트북 구독 서비스와 SaaS의 토큰 예산 선불제(유틸리티 과금)가 같은 방향을 가리킨다 — 하드웨어와 소프트웨어 모두 "소유"에서 "구독"으로 이동하고 있다.

**4. 스킬 생태계의 플랫폼 전쟁과 표준화 경쟁**: Anthropic의 skill-creator eval 프레임워크, OpenAI의 Codex Skills Catalog(11,500 스타), 커뮤니티의 K-Skill, Obsidian CEO의 직접 스킬 제작이 동시에 진행되고 있다. Goobong Jeong의 "스킬이 코드에서 스펙으로 진화"한다는 분석과 Jeongmin Lee의 3계층 스펙 파일 구조가 같은 방향을 가리킨다 — 궁극적으로 description이 곧 skill이 되는 선언적 프로그래밍 패러다임이 에이전트 생태계에서도 재현되고 있다.

**5. 데이터 접근 확대 vs 프라이버시**: GitHub의 Copilot 학습 데이터 수집, Windows 11의 Microsoft 계정 강제, Palantir의 "연구 외 목적" 조항, 인스타그램/유튜브 아동 중독 설계 판결이 모두 "기술 기업의 데이터 접근 확대"라는 하나의 축 위에 놓인다. 동시에 Anthropic의 자율 무기/대량 감시 레드라인, ATLAS의 완전 셀프호스팅, Project NOMAD의 오프라인 컴퓨팅이 "데이터 주권"이라는 반대 축을 형성한다.
