## Claude Code 소스코드 유출 사태

오늘 수집된 콘텐츠 중 가장 많은 글이 집중된 주제다. npm 패키지 설정 오류로 Anthropic의 핵심 제품 코드베이스가 전량 노출되었고, 기술 분석, 오픈소스 프로젝트, 바이럴 현상, 법적 대응, 만우절 장난까지 다층적으로 전개되었다.

### 유출 경위와 규모
[출처 - Lior Alexander / LinkedIn](https://www.linkedin.com/feed/update/urn:li:activity:7444818234236891136/)

Claude Code의 npm 패키지 `anthropic-ai/claude-code` 버전 2.1.88에 잘못 포함된 `.map` 파일(cli.js.map, 59.8MB)을 통해 Anthropic의 R2 버킷에서 전체 TypeScript 코드베이스를 직접 다운로드할 수 있는 링크가 노출되었다.

유출 규모:
- **1,900개 파일**, **512,000줄 이상**의 TypeScript 코드
- **QueryEngine.ts**: 약 46,000줄 — 핵심 LLM API 엔진
- **멀티 에이전트 오케스트레이션**: Claude가 서브 에이전트를 생성하는 구조, 내부적으로 **"swarms"**라고 부름
- **양방향 IDE 브리지**: VS Code와 JetBrains를 JWT 인증 채널로 연결
- **KAIROS**: 미출시 자율 데몬 모드 — Claude가 항상 켜진 백그라운드 에이전트로 동작
- **"Undercover Mode"**: Anthropic 내부 정보가 git 커밋에 유출되는 것을 방지하는 시스템

Anthropic은 이후 버전을 2.1.88에서 2.1.87로 롤백했다.

분석된 주요 아키텍처 요소:

- **감소 수익 감지(Diminishing Returns Detection)**: 무한 루프를 자동 차단하는 메커니즘

미출시 기능:

- **멀티 에이전트 Coordinator**
- **선제적 Kairos 모드**: 항상 켜져 있는 백그라운드 에이전트 데몬

### Claude Code 동작 방식 완전 해부
[출처 - GeekNews](https://news.hada.io/topic?id=28062)

핵심 아키텍처는 **ReAct 패턴** 기반의 반복적 Agentic Loop로, "읽고 → 생각하고 → 도구 쓰고 → 결과 보고 → 반복"하는 6단계 프로세스를 따른다.

**Agentic Loop의 6단계:**
1. 사용자가 터미널에 메시지 입력
2. 시스템 프롬프트에 현재 날짜, git 상태, CLAUDE.md, 도구 목록 포함하여 문맥 조립
3. Anthropic API 호출로 `tool_use` 블록 생성
4. allow/ask/deny 중 하나로 도구 실행 여부 결정 (권한 검증)
5. 파일 조작, 셸 명령 등 도구 실행 후 결과 반환
6. 도구 호출이 없을 때까지 루프 반복

**컨텍스트 관리:** 모든 API 호출 앞에 시스템 컨텍스트(현재 브랜치, 최근 5개 커밋, git status)와 사용자 컨텍스트(CLAUDE.md 메모리 파일, 현재 날짜)가 붙는다.

**권한 모델:** 읽기 전용 도구(Read, Glob, Grep)는 모든 모드에서 자동 승인되며, `bypassPermissions` 모드는 모든 체크를 스킵하고, `acceptEdits` 모드는 파일 편집만 자동 승인한다.

**서브에이전트(Task 도구):** Claude는 AgentTool로 격리된 서브에이전트를 생성할 수 있으며, 독립적인 루프를 실행하고 결과를 부모 에이전트에 반환한다. 대화는 `~/.claude/`에 JSON으로 저장되며, `--resume`으로 재개 가능하고, 긴 대화는 주기적으로 요약(컴팩션) 처리된다.

### 바이럴 메커니즘 분석 — 11시간 만에 역대 최고 속도 30K 스타
[출처 - Zoon Chang / LinkedIn](https://www.linkedin.com/feed/update/urn:li:activity:7444871503797571584/)

Zoon Chang이 Jin Hyung Park (Sigrid Jin)의 GitHub 패키징 프로젝트가 터진 바이럴 메커니즘을 분석했다.

바이럴 타임라인:
1. Claude Code 소스코드 유출 소식이 트위터에 올라옴
2. **1시간 안에** Sigrid Jin이 GitHub으로 패키지화하여 업로드
3. 트위터에서 11시간 만에 **185만 조회수** 달성
4. 매 시간마다 **GitHub 스타 5,000개** 추가, Fork는 그 이상
5. 패키지를 법적으로 덜 위험하게 재패키징
6. 팀 매니징을 시작하여 브랜딩 및 세계관 확장
7. "역사상 가장 빠른 30K(이후 50K) 돌파"라는 사실 자체를 콘텐츠로 만들어 2차 바이럴 설계

핵심 분석: **Star 5만은 즐겨찾기**지만, **Fork가 5.6만** — Star보다 Fork가 많다. 5.6만 명이 코드를 가져가서 뭔가를 시작했다는 의미로, 관심이 아니라 **참여**다. "역대 최고 속도 30K 돌파"라는 사실 자체를 다시 콘텐츠로 만드는 **재귀적 바이럴 구조**가 작동했다.

### Apple Silicon M4/M5 칩에서 외부 4K HiDPI 해상도 제한
[출처 - GeekNews](https://news.hada.io/topic?id=28068)

M4 및 M5 칩 기반 Mac에서 외부 4K 모니터의 3840x2160@2x HiDPI 모드가 더 이상 작동하지 않는 문제가 보고되었다. 하드웨어 성능 부족이 아니라 **Display Coprocessor(DCP) 펌웨어의 구조 변경**이 원인이다.

M5 Max에서 sub-pipe 구조 변경으로 단일 스트림 파이프의 예산이 6720픽셀로 축소되어 4K HiDPI에 필요한 7680x4320 백킹스토어 생성이 불가능하다. Display Override Plist 수정, EDID 패치, IOKit Registry 직접 수정 등을 모두 시도했으나 실패. 유일한 임시 해결책은 SkyLight의 비공개 `SLVirtualDisplay` API를 이용한 가상 디스플레이 + 하드웨어 미러링이나 불안정하다. Apple Feedback **FB22365722**가 제출된 상태다. 외부 4K 모니터를 사용하는 개발자와 디자이너에게 직접적인 영향을 준다.

## 금융 시장 사이클과 AI 투자

### 골드만삭스 전 CEO Lloyd Blankfein 인터뷰: "사이클 후반부에 접어들었다"
[출처 - 비즈니스캔버스 / YouTube](https://www.youtube.com/watch?v=BcVUPUOi8q4)

2018년 골드만삭스 CEO에서 퇴임한 Lloyd Blankfein이 은퇴 생활, AI 시각, 사모신용 시장 위험, 파트너십 문화, 기업과 정치의 관계까지 폭넓게 다룬 약 7,000단어 분량의 인터뷰다.

**AI에 대한 견해: "대부분은 parlor trick이지만, 투자하는 사람들은 자기 돈을 걸고 있다"**

Blankfein은 AI를 주로 구글 검색의 대안으로 사용하고 있다. "When I use Google, I get a bibliography. When I use chat or one of the other AI, I get an essay that purports to be the answer." AI의 에세이형 답변은 정확성을 검증하기 어렵다고 지적했다.

AI 투자에 대해서는 하이퍼스케일러들이 연간 1,000억 달러에 가까운 금액을 투자하고 있다고 언급하며, "How many countries can invest in R&D for over $100 billion a year?"라고 그 규모를 국가 단위와 비교했다. 다만 창업자들이 자기 자산을 걸고 있다는 점에서 판단을 존중하면서도, "Even if it's perfect and right in every way, not every company doing it will be a winner"라고 선별의 불가피성을 강조했다. 단기적으로는 오히려 고용이 늘어난다는 전망이다.

**사모신용(Private Credit) 시장의 위험**

Jamie Dimon의 경고와 Marathon의 부도율 상승 예측 상황에서, 세 가지 구조적 문제를 짚었다. 첫째, 비유동적이고 불투명한 자산이라 정확한 가격 확인이 극히 어렵다. 둘째, 장기간 호황으로 규율이 이완되었다. 셋째, 《대부》를 인용하며 "10년마다 이런 일이 있어야 쌓인 나쁜 피를 정리할 수 있다"고 시장 조정의 불가피성을 역설했다. 핵심 판단: "I don't know if we're in the absolute end of the cycle, but we're getting close."

**퇴직연금에 사모자산 편입 반대**

### INSID3: 학습 없이(Training-Free) DINOv3만으로 분할 SOTA 달성
[출처 - HuggingFace Papers](https://huggingface.co/papers/2603.28480)

In-context segmentation(ICS) 과제에서 단일 자기지도 학습 백본(DINOv3)만으로, 어떠한 감독 학습이나 보조 모델 없이 의미적 매칭과 분할을 동시에 수행하는 INSID3를 제안했다.

핵심 기술 세 가지: (1) DINOv3 특징 공간의 **위치 편향 제거** — SVD로 위치 부분공간을 추정한 뒤 직교 보공간에 투영. 학습 없이 단일 행렬 곱셈으로 수행. (2) 응집 클러스터링으로 **세밀한 클러스터링** — 단일 임계값 tau=0.6으로 세분화 수준 제어. (3) **시드-클러스터 선택 및 집계** — 역방향 대응으로 후보를 찾고 교차 이미지 유사도와 자기 유사도를 결합.

시맨틱 분할 6개 데이터셋에서 평균 55.1% mIoU(GF-SAM 47.6% 대비 +7.5%), 파트 분할 평균 44.6% mIoU(+4.2%), 개인화 분할 67.0% mIoU(GF-SAM 54.1% 대비 +12.9%). **304M 파라미터**로 SAM 기반 파이프라인(945M)의 3분의 1 크기에 불과하다.

### ChartNet: 150만 규모 차트 이해 멀티모달 데이터셋

- 합성 학습 데이터가 실세계 공개 벤치마크에 효과적으로 전이

### STRIDE: 스트리밍 비디오에서 "언제 말할지" 결정하는 모델
[출처 - HuggingFace Papers](https://huggingface.co/papers/2603.27593)

UIUC와 KAIST 연구진이 스트리밍 비디오에서의 능동적 활성화를 **masked diffusion 기반 시퀀스 디노이징**으로 재정의한 STRIDE를 제안했다. 기존 프레임 단위 이진 분류의 "깜빡임" 현상을 해결한다.

아키텍처: (1) 활성화 모델(Qwen3-VL-2B 기반 경량 모듈)이 스트림을 모니터링하며 응답 시점을 결정, (2) 하위 Video-LLM(동결)이 활성화 시 응답 생성. 세 가지 구조화된 마스킹 전략과 선택적 재마스킹, 점진적 디노이징으로 추론 시 시간적 일관성을 확보한다.

OVO-Bench에서 Forward Active Responding 46.30→59.70(+13.4점), StreamingBench에서 46.84→59.29(+12.45점), ET-Bench TVG F1 35.7→62.8(+27.1점). 새 프레임당 113ms 지연으로 베이스 모델 응답 시간의 7%에 불과하며, 다양한 백본에 플러그인 방식으로 적용 가능하다. 학습 데이터는 8개 데이터셋 151,736개 비디오, 687,278개 세그먼트를 활용했다.

## 교차 분석

### 1. "큰 모델보다 나은 활용"이 공통 메시지
논문의 INSID3(학습 없이 304M으로 SOTA), ChartNet(2B/7B로 GPT-4o 초과), 뉴스의 1-Bit Bonsai(1비트로 16비트 수준 정확도), 새의 두뇌(작지만 뉴런 밀도가 높은 효율적 인지)가 모두 "더 큰 것보다 더 나은 활용"을 증명한다. Blankfein의 AI 견해("parlor trick이지만 투자자들은 자기 돈을 걸고 있다")는 이 효율성 트렌드와 거대 자본 투자 사이의 간극을 보여준다.

### 2. "사이클 후반부" 경고와 $852B 기업가치의 긴장
Blankfein이 사모신용, 퇴직연금, AI 투자를 관통하며 반복한 "late cycle" 프레임은 같은 날 보도된 OpenAI의 $852B 기업가치와 직접적으로 대조된다. 둘 다 AI 산업을 다루지만, 한쪽은 "절정에 가까워지고 있다"는 경고이고 다른 쪽은 "전례 없는 규모의 자본이 몰리고 있다"는 사실이다. Blankfein의 "Even if it's perfect, not every company doing it will be a winner"라는 발언은 OpenAI의 기업가치에도 적용될 수 있는 프레임이다.

### 3. 유출 사건이 드러낸 "열림 vs 닫힘"의 다층적 긴장
Claude Code 소스코드 유출은 단순한 보안 사고를 넘어 플랫폼 개방성 논쟁과 공명한다. 카카오톡의 닫힌 정책이 계정 삭제로 이어진 반면, MS Teams는 OpenClaw 연동을 적극 추진하고, 유출된 Claude Code는 하루 만에 50K 스타 오픈소스 프로젝트로 변모했다. Android 개발자 인증 제도와 Apple의 HiDPI 제한도 같은 축의 다른 지점이다. AI 시대의 경쟁은 "인터페이스를 붙잡는 것이 아니라 자동화를 감당할 프로토콜을 설계하는 것"이라는 JungHoon Ghae의 전망이 이 전체 흐름을 관통한다.

### 4. AI 품질 논쟁의 두 얼굴
"AI 이전 글쓰기 시대가 그리워진다"(비관)와 "Slop Is Not Necessarily the Future"(낙관)는 동일 현상의 양면이다. 전자는 인간의 창의적 목소리 상실을, 후자는 경제적 인센티브에 의한 자연 교정을 주장한다. Blankfein의 AI 견해("bibliography vs essay")는 이 논쟁에 제3의 시각을 더한다 — AI 답변의 정확성 검증 문제. 논문 카테고리의 ChartNet은 이 논쟁에 데이터로 답한다 — 고품질 정렬 데이터가 있으면 소형 모델도 대형 모델을 넘길 수 있다.

### 5. "인간이 병목" — 전문가에서 도메인 사용자로의 전환
카파시의 "내가 병목이 되어선 안 된다", dev._pill의 "AX팀은 Enabler", Ruben Hassid의 "코딩을 배울 필요 없다", 거지맵의 "비개발자가 11일 만에 4.5만 사용자"가 같은 방향을 가리킨다. AI 도구의 가치는 전문가가 더 잘하는 것이 아니라, 비전문가가 할 수 있게 되는 것에 있다. 이는 STRIDE 논문의 모듈식 설계(다양한 백본에 플러그인)나 INSID3의 단일 백본 다목적 활용과도 구조적으로 유사하다 — 핵심은 접근성과 범용성이다.
