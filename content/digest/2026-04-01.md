### axios npm 공급망 RAT 공격 — 즉시 대응 필요
[GeekNews](https://news.hada.io/topic?id=28047)

주간 다운로드 **1억 회 이상**의 JavaScript HTTP 클라이언트 라이브러리 **axios**의 npm 계정이 침해되어, 2026년 3월 30~31일에 걸쳐 크로스플랫폼 RAT(원격 접근 트로이목마) 드로퍼가 포함된 악성 버전이 배포됐다. 발견·분석은 StepSecurity가 수행했다.

**악성 버전:** `axios@1.14.1`, `axios@0.30.4` — 약 3시간 노출 후 npm에서 삭제됐다.

**공격 타임라인:**

| 시각 (UTC) | 이벤트 |
|---|---|
| 03-30 05:57 | `plain-crypto-js@4.2.0` (깨끗한 디코이) 배포 — 계정 이력 확보용 |
| 03-30 23:59 | `plain-crypto-js@4.2.1` (악성 드로퍼 포함) 배포 |
| 03-31 00:21 | `axios@1.14.1` 악성 버전 배포 |
| 03-31 01:00 | `axios@0.30.4` 악성 버전 배포 (39분 간격) |
| 03-31 ~03:15 | npm에서 두 버전 삭제 |

**공격 메커니즘:** axios 자체 코드에는 악성 코드가 없다. `package.json`에 `plain-crypto-js@^4.2.1`만 의존성으로 추가했고, `npm install` 시 `postinstall` 훅이 자동 실행되면서 RAT 드로퍼가 2초 이내에 C2 서버(`http://sfrclak.com:8000/6202033`)와 통신한다.

**플랫폼별 페이로드:**
- **macOS:** AppleScript로 `/Library/Caches/com.apple.act.mond`(Activity Monitor Daemon 위장)에 2단계 페이로드 설치
- **Windows:** PowerShell을 `%PROGRAMDATA%\wt.exe`(Windows Terminal 위장)에 복사, VBScript로 은닉 실행
- **Linux:** curl로 Python RAT를 `/tmp/ld.py`에 다운로드 후 nohup 백그라운드 실행

**반포렌식 기법:** 실행 후 `setup.js`가 자신을 삭제하고, 악성 `package.json`을 사전 준비된 클린 스텁으로 덮어쓴다. `npm list`는 `plain-crypto-js@4.2.0`으로 보고하므로 버전 번호로는 탐지 불가. **디렉터리 존재 여부**가 유일한 지표다.

**즉시 실행 항목:**
```bash
# 1. 악성 버전 확인
npm list axios 2>/dev/null | grep -E "1\.14\.1|0\.30\.4"

# 2. plain-crypto-js 존재 확인 (존재하면 드로퍼 실행됨)
ls node_modules/plain-crypto-js 2>/dev/null && echo "POTENTIALLY AFFECTED"

# 3. 플랫폼별 RAT 아티팩트 확인
# macOS
ls -la /Library/Caches/com.apple.act.mond
# Linux
ls -la /tmp/ld.py
# Windows
dir "%PROGRAMDATA%\wt.exe"

# 4. 안전 버전으로 교체
npm install axios@1.14.0
rm -rf node_modules/plain-crypto-js
npm install --ignore-scripts
```

**CI/CD 보안 권고:** `npm ci --ignore-scripts`를 표준 정책으로 적용해 `postinstall` 훅의 자동 실행을 방지할 것. C2 도메인(`sfrclak.com`, IP: 142.11.206.73) 네트워크 차단 권장. 정상 axios 1.x 릴리스는 GitHub Actions OIDC로 배포되지만, `1.14.1`은 장기 npm 토큰으로 수동 배포됐다 — OIDC Trusted Publisher 위반이 핵심 탐지 신호다.

### Claude Code npm 소스맵 유출 — 51만 줄 TypeScript 공개
[GeekNews](https://news.hada.io/topic?id=28059) | [HackerNews](https://github.com/chatgptprojects/claude-code) | [최초 보고 트윗](https://twitter.com/Fried_rice/status/2038894956459290963)

2026년 3월 31일, Chaofan Shou(@Fried_rice)가 Claude Code CLI 소스코드가 npm 패키지 `@anthropic-ai/claude-code` v2.1.88에 `cli.js.map` 소스맵 파일 형태로 포함돼 있다는 사실을 공개했다. Anthropic이 `.npmignore`에서 `.map` 파일을 제외하지 않은 배포 실수다. 해킹이 아닌 운영 실수로, 유출된 것은 모�� 가중치가 아닌 CLI TypeScript 소스코드다.

추출 방법:
```bash
npm pack @anthropic-ai/claude-code@2.1.88
tar -xzf anthropic-ai-claude-code-2.1.88.tgz
cd package
node unpack.mjs  # sourcesContent에서 파일 추출
```

**커뮤니티에서 발굴한 주요 인사이트 10가지 (utkarsh_aryan 정리):**

1. **Undercover Mode (은밀 오픈소스 기여):** 리포지토리가 내부 허용 목록에 없으면 자동 활성화되며, `Co-Authored-By` 줄이나 AI 언급 없이 코드를 커밋하도록 설계됐다. Opt-out 불가.

2. **미래 모델 코드네임:** `opus-4-7`, `sonnet-4-8` 문자열 등장. 코드네임 **Capybara**(capybara, capybara-fast, capybara-fast[1m]), 이전 Opus의 코드네임은 **Fennec**.

3. **캐싱 우선 설계:** `getFeatureValue_CACHED_MAY_BE_STALE()`, `DANGEROUS_uncachedSystemPromptSection()` 같은 명명 규칙이 레이턴시 최우선 설계를 반영.

4. **YOLO 분류기:** ML 기반 자동 권한 결정 시스템. 대화 트랜스크립트를 분석해 툴 호출을 사용자 확인 없이 자동 승인. 완전 자율 에이전트로 가는 경로가 이미 구현.

5. **Dream 시스템:** 세션 간 오프라인 처리 기능. 마지막 dream 이후 24시간 경과, 세션 5회 이상 등의 트리거 조건. 장기 메모리 기능.

6. **안전 경계의 개인 소유:** `// IMPORTANT: DO NOT MODIFY THIS INSTRUCTION WITHOUT SAFEGUARDS TEAM REVIEW` — 보안 경계에 특정 개인 이름이 하드코딩.

7. **토큰 탈취 방지:** 프록시가 `prctl(PR_SET_DUMPABLE, 0)`을 사용해 ptrace에 의한 힙 메모리 읽기를 차단.

8. **클라이언트 어태스테이션:** Bun HTTP 스택의 DRM 유사 메커니즘. 단, npm 설치 버전에서는 미적용.

9. **숨겨진 완성 기능들:** KAIROS(장기 기억), ULTRAPLAN(고급 계획), Buddy 모드, Coordinator Mode, Agent Teams, Dream, YOLO 분류기 등이 컴파일 타임 플래그로 출시 빌드에서 제외.

10. **Claude Code 자체가 API 테스트베드:** `redact-thinking`, `afk-mode`, `advisor-tool`, `task-budgets` 같은 미공개 API 기능 참조가 존재. 3~6개월 후 공개 API에 등장할 기능의 미리보기.

**텔레메트리:** 사용자 욕설(`matchesNegativeKeyword`)과 "continue", "keep going" 입력을 추적하는 텔레메트리가 존재. 사용자 불만 측정 목적으로 추정.

**SNS 반응 종합:** Threads에서는 "AGI의 비밀이 80만 줄의 TypeScript 조건문이었다"(choi.openai), "코딩 보조 도구가 아니라 AI 에이전트 OS"라는 평가가 다수. 유출 후 수 시간 만에 전용 분석 문서 사이트, 한국어 번역본, 18개 카테고리 분석 보고서가 등장했다. 커밋 메시지가 "asdf"라는 황당한 발견(ai.hyeop)도 화제.

**역설:** AI가 내부 정보를 외부에 흘리는 것을 막는 Undercover Mode를 가진 회사의 소스코드가 `.npmignore` 한 줄 누락으로 유출됐다(jisang0914).

[X @vineetwts](https://x.com/vineetwts/status/2038911973975601275)

유출 코드베이스를 분석해 파이프라인·동작 방식·컨텍스트·메모리 처리 등을 정리한 문서가 공개됐다. 문서: https://www.mintlify.com/VineeTagarwaL-code/claude-code/concepts/how-it-works. 핵심 발견: Claude Code는 고정 문자열이 아닌 모듈식 section-builder 함수를 통해 런타임에 동적으로 조립되는 다층 프롬프트 아키텍처를 사용하며, 경계 마커가 전역 캐시 가능한 prefix와 세션별 suffix를 구분해 API 호출 전반에서 프롬프트 캐싱이 가능하다.

52만 줄을 18개 카테고리로 분석한 결과(darkest_alex Threads): Undercover Mode, 텔레메트리 732개, 이메일 평문 전송, 킬스위치 이름이 "난독화"이며 코드 주석에 "Mangled name"이라고 직접 기재. 코드 품질은 5000줄짜리 단일 파일, 순환 의존성, 타입 검증 없는 JSON 파싱 등도 발견됐다.

시스템 아키텍처는 3구성 요소: 인간(CLI Interface) — 뇌(Query Engine, 의도 분석·도구 선택) — 손과 발(Tools, 파일 읽기·쉘 명령어)(ai_tutor_lab Threads).

### AX를 고민하는 대표들에게 — 채널톡 CAIO 이경훈
[YouTube — EO Korea](https://www.youtube.com/watch?v=fPH4-YXirsc)

채널코퍼레이션(채널톡) CAIO가 사내 AI 트랜스포메이션 과정에서 겪은 시행착오와 핵심 인사이트를 공유한 영상이다.

**실패 1: 전원 교육 + 툴 제공** — 매일 저녁 MCP 스터디를 제공했으나 참여율 저조. 결론: 동일 콘텐츠를 동일 방식으로 제공하는 것은 효율적이지 않다.

**실패 2: AI 전문가 소수 팀** — 세 가지 문제: 현장 지식 부재("프롬프트를 줄 수가 없는 거라는 깨달음"), 스케일 문제(요청이 쏟아지며 동기부여 급락), 경쟁력 한계("경쟁사 200명이 다 AI를 잘 쓴다면?").

**해결책: 도메인별 챔피언 프로그램** — 1기 약 3개월, 5~6명 선발. 심리적 장벽을 낮추고 의지를 형성한 뒤 Claude Code 등 강력한 툴을 전사 제공. "1년 전에 바로 Claude Code 쓰세요라고 했다면 안 됐을 것."

**핵심 개념: 암묵지(Tacit Knowledge) 추출** — 인터넷에도 사내 문서에도 없고 직원 머릿속에만 있는 지식을 데이터화하는 것이 AI 시대 기업 핵심 역량. 채널톡 사례: 고객사 상담사의 수년간 채팅 이력을 분석해 SOP 초안을 생성 → 상담사가 확인·수정 → 암묵지가 데이터로 변환. 이 암묵지는 빅테크도 경쟁사도 가질 수 없는 해자(moat).

**B2B SaaS 시장 붕괴 전망:** SaaS 원가율 10~15%. AI로 제작이 쉬워지면서 "굳이 비싼 거 살 이유가 없다." 단, 양극화: 주가 하락 SaaS와 Decagon·Sierra 같은 수조 원 밸류에이션 에이전트 스타트업 공존. 차이의 핵심: 툴 시장(축소) vs 인건비 시장(폭등). 한국 SaaS 1~2조 원 규모 대비 콜센터 BPO 시장만 10조 원.

**AI 시대 개인 처우 문제:** "디자이너가 10명분을 한다. 10명분 임금을 주는가? 안 줄 것." → AI 잘 쓰는 인재가 보상하는 회사나 직접 창업으로 이탈. 보상 체계와 성장 환경이 인재 유지의 필수 조건.

**조직 변혁 방법론: 역할에서 워크플로로** — "편집자를 고용해야 한다" → "편집자가 하는 5가지 행동이 무엇인가? 각각을 워크플로로 만들 수 있는가?" 채용 때마다 4~10가지 구체적 작업을 나열하고, 헤드카운트 대신 워크플로로 존재할 수 있는지 검토.

**BYOS/BYOA(Bring Your Own Software/Agent):** 개인이 자신만의 에이전트 시스템을 보유하고 "나는 마케팅 부서 전체입니다"라고 접근하는 구조. Anthropic이 마케팅 부서를 1명이 에이전트로 운영한다는 사례 언급.

**AI 훈련 핵심:** 12개의 깨지 않을 규칙과 16개의 글 샘플을 제공하면 결과물이 5배. 사람과 1년 반 걸릴 100번의 피드백 사이클이 AI와는 100분.

**바벨 전략:** 고위험 극단(AI 퍼스트 비즈니스) + 안전 극단(절대 변하지 않을 것: 헬스케어, 엔터테인먼트). 예측 지표: "포르노 산업이 먼저 채택한 기술이 나중에 주류화된다."

**즉시 실행 계획:** (1) 매일 작업을 가장 세분화된 단위로 나열 (2) 각 태스크를 AI에 "자동화 단계?"로 질문 (3) 막히면 스크린샷을 AI에 붙여 넣고 "지금 뭘 해야 하나?" 질문.

## 교차 분석

### 1. npm 생태계가 이틀 연속 양방향으로 무너졌다

Claude Code 소스맵 유출(실수에 의한 소스 노출)과 axios RAT 공급망 공격(의도적 악성코드 주입)이 같은 주말에, 같은 npm 인프라에서 발생했다. 하나는 `.npmignore` 한 줄 누락, 하나는 `postinstall` 훅 악용. 두 사건은 npm이라는 단일 레지스트리에 대한 신뢰 모델이 양쪽 방향(유출과 침투) 모두에서 구조적으로 취약함을 동시에 증명했다.

### 2. 도메인 지식이 AI 시대의 진짜 해자다

수술 AI 논문(VLM 235B가 YOLO 26M에 진다), Merve Noyan("소형 특화 모델이 과소평가"), 이경훈("암묵지 데이터화가 기업 해자"), AX Champions("도구보다 문제 정의가 먼저"), Hormozi("정확한 아웃풋 정의 능력"), 홍민지("문제와 목적에서 시작하라"), 알렉스 카프("체화된 지능과 비선형 사고")가 모두 같은 결론을 가리킨다: 범용 스케일링은 한계가 있고, 현장의 깊은 이해가 AI를 의미 있게 만드는 핵심이다.

### 3. 에이전트 하네스가 AI 개발의 다음 전선이 되고 있다

Claude Code 유출 코드가 "코딩 보조가 아니라 AI 에이전트 OS"로 평가받고, claw-code가 수 시간 만에 30K 스타를 달성하고, Graph Workflow가 9.59배 지속·35% 품질 향상을 보이고, OmC/OmX가 팀 오케스트레이션의 철학적 근거를 제시하고, KAT-Coder-V2가 10개+ 스캐폴드를 통합하는 KwaiEnv를 구축한 것은 모두 같은 방향이다: "모델 성능"에서 "모델을 어떻게 배선·조율·운영하는가"로 경쟁의 축이 이동하고 있다.

### 4. AI 채택 저항의 극복 — 심리적 장벽 > 기술적 장벽

AX Champions의 "심리적 준비 없이 툴만 던지면 효과 없다", 이경훈의 "1년 전에 바로 Claude Code 쓰세요라고 했다면 안 됐을 것", Hormozi의 "현상 유지 편향이 안전 우려보다 큰 원인", Merve의 "해결하고 싶은 실제 문제를 먼저 찾아라"가 공통으로 지적하는 것: AI 채택의 진짜 장벽은 기술이 아니라 심리와 동기다.

### 5. 비개발자가 만드는 결과물이 더 흥미롭다는 역설

UI 디자이너의 벚꽃 지도, 마케터의 광고 자동화, 백엔드 개발자의 Steam 게임, 재무팀의 자동 정산. "전문 프로그래머가 아닌 분들의 작업물이 더 재미있다"는 관찰은 도메인 지식 + AI 도구의 조합이 순수 기술 능력보다 더 큰 가치를 만들어낸다는 증거다. 이것은 2번(도메인 지식이 해자)과 4번(심리적 장벽 극복)의 자연스러운 결과물이다.
